# 经济奖励系统测试指南

## 系统概述
升格怪物Up!Mobs现在集成了经济奖励系统，玩家击杀进化怪物可以获得游戏币奖励。

## 前置要求
1. **Vault插件**：必须安装Vault插件
2. **经济插件**：必须安装一个经济插件（如EssentialsX、CMI等）
3. **权限插件**：可选，如果需要权限控制

## 奖励规则

### 1. 进化怪物奖励
- **第一阶段**：基础10游戏币 × 随机0.7-1.3倍
- **第二阶段**：20游戏币 × 随机0.7-1.3倍
- **第三阶段**：40游戏币 × 随机0.7-1.3倍
- **第四阶段**：额外50游戏币 × 随机0.8-1.2倍

### 2. 升格怪物奖励
- **自然刷新升格怪物**：15游戏币 × 随机0.75-1.25倍

### 3. 奖励限制
- 单次击杀最大奖励：200游戏币（可配置）
- 最小奖励：5游戏币（可配置）
- 可配置是否需要权限

## 测试步骤

### 步骤1：环境准备
1. 安装Vault插件
2. 安装经济插件（如EssentialsX）
3. 安装升格怪物Up!Mobs插件

### 步骤2：配置检查
检查配置文件中的经济奖励设置：
```yaml
economy_rewards:
  enabled: true
  base_amount: 10.0
  stage_multiplier: 2.0
  stage4_bonus: 50.0
  upgraded_amount: 15.0
```

### 步骤3：测试进化怪物奖励
1. 让怪物攻击玩家3次以上，触发进化
2. 击杀进化后的怪物
3. 检查玩家是否收到游戏币奖励
4. 检查控制台日志是否有奖励记录

### 步骤4：测试第四阶段怪物奖励
1. 等待自然刷新第四阶段怪物（8%几率）
2. 击杀第四阶段怪物
3. 检查玩家是否收到额外奖励

### 步骤5：测试升格怪物奖励
1. 等待自然刷新升格怪物（30%几率）
2. 击杀升格怪物
3. 检查玩家是否收到奖励

## 故障排除

### 问题1：没有收到奖励
1. 检查Vault插件是否安装
2. 检查经济插件是否安装并配置
3. 检查配置文件中的`economy_rewards.enabled`是否为true
4. 检查玩家是否有权限（如果配置了权限要求）

### 问题2：奖励金额不正确
1. 检查配置文件中的奖励金额设置
2. 检查随机因子是否正常工作
3. 检查最大奖励限制

### 问题3：没有奖励消息
1. 检查配置文件中的`message_enabled`是否为true
2. 检查消息格式配置
3. 检查玩家聊天设置

## 配置示例

### 基础配置
```yaml
economy_rewards:
  enabled: true
  base_amount: 10.0
  stage_multiplier: 2.0
  min_amount: 5.0
  stage4_bonus: 50.0
  upgraded_amount: 15.0
  message_enabled: true
  max_reward_per_kill: 200.0
  require_permission: false
```

### 高奖励配置（适合高经济服务器）
```yaml
economy_rewards:
  enabled: true
  base_amount: 50.0
  stage_multiplier: 3.0
  min_amount: 20.0
  stage4_bonus: 200.0
  upgraded_amount: 75.0
  max_reward_per_kill: 1000.0
```

### 低奖励配置（适合生存服务器）
```yaml
economy_rewards:
  enabled: true
  base_amount: 5.0
  stage_multiplier: 1.5
  min_amount: 2.0
  stage4_bonus: 20.0
  upgraded_amount: 8.0
  max_reward_per_kill: 50.0
```

## 性能考虑
1. 经济奖励系统对性能影响很小
2. 只在怪物死亡时进行计算
3. 奖励计算是轻量级的
4. 消息发送使用异步处理

## 安全考虑
1. 可配置权限要求
2. 可配置最大奖励限制
3. 防止奖励滥用
4. 完整的日志记录
