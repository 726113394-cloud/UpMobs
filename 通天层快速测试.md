# 通天层快速测试

## 🚀 快速测试步骤

### 1. 环境准备
```bash
# 确保插件已加载
/upmobs help

# 检查挑战塔系统
/upmobs tower list

# 检查经济系统
/upmobs get economy
```

### 2. 创建测试挑战塔（如果还没有）
```bash
# 获取区域选择工具
/upmobs tower tool

# 选择区域（左键设置点1，右键设置点2）
# 创建挑战塔
/upmobs tower create test_tower "测试挑战塔"
```

### 3. 快速通关挑战塔（测试用）
```bash
# 加入挑战塔
/upmobs tower join test_tower

# 开始挑战（需要其他玩家或管理员权限）
# 快速完成18层（测试环境可以调整配置）
```

### 4. 测试通天层
```bash
# 查看通天层帮助
/upmobs tower sky

# 开始通天层挑战
/upmobs tower sky start test_tower

# 查看状态
/upmobs tower sky status

# 结束挑战
/upmobs tower sky end
```

## 🔧 快速配置调整（测试用）

### 1. 降低通关要求（测试用）
```bash
# 修改通天层解锁要求为"none"（测试用）
/upmobs set sky_tower.unlock_requirement none
```

### 2. 缩短时间限制（测试用）
```bash
# 缩短通天层时间到2分钟
/upmobs set sky_tower.time_limit.minutes 2

# 缩短警告时间
/upmobs set sky_tower.time_limit.warning_seconds 30
```

### 3. 减少怪物数量（测试用）
```bash
# 减少怪物数量到3只
/upmobs set sky_tower.monster.base_count 3

# 减少刷新延迟
/upmobs set sky_tower.monster.refresh_delay_seconds 2
```

### 4. 降低难度（测试用）
```bash
# 降低难度增强
/upmobs set sky_tower.difficulty.increase_per_minute 0.05

# 禁用环境变化
/upmobs set sky_tower.difficulty.environmental_changes false
```

## 🐛 常见问题排查

### 问题1：无法开始挑战
```bash
# 检查配置
/upmobs get sky_tower.enabled

# 检查解锁要求
/upmobs get sky_tower.unlock_requirement

# 检查玩家是否已在挑战中
/upmobs tower sky status
```

### 问题2：怪物没有生成
```bash
# 检查怪物配置
/upmobs get sky_tower.monster.base_count
/upmobs get sky_tower.monster.types

# 检查挑战塔区域
/upmobs tower info test_tower
```

### 问题3：奖励没有发放
```bash
# 检查经济系统
/upmobs get economy.enabled

# 检查奖励配置
/upmobs get sky_tower.rewards.base_per_kill

# 检查玩家余额
/money TestPlayer
```

### 问题4：时间限制无效
```bash
# 检查时间配置
/upmobs get sky_tower.time_limit.minutes

# 检查计时器显示
/upmobs get sky_tower.time_limit.show_timer
```

### 内存使用
- 每个通天层实例：约1-2KB
- 怪物管理：约100字节/只
- 任务调度：轻量级

### 建议
- 同时运行的挑战数量：建议不超过10个
- 怪物数量：建议10-20只
- 刷新延迟：建议3-5秒
