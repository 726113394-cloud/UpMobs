# 通天层系统测试指南

## 🏔️ 系统概述
通天层是一个无尽挑战模式，玩家在通关挑战塔所有18层后可以手动选择进入。这是一个固定10只第四阶段升格怪物的无尽挑战，玩家死亡或10分钟时间到即结束。

## 🎯 核心特性

### 1. 触发条件
- **通关要求**：必须通关挑战塔所有18层
- **手动进入**：通关后使用命令手动选择进入
- **单人挑战**：每个玩家独立进行挑战

### 2. 怪物系统
- **固定数量**：始终保持10只第四阶段升格怪物
- **死亡刷新**：怪物死亡后5秒自动刷新
- **怪物类型**：第四阶段僵尸、骷髅、苦力怕、蜘蛛、末影人
- **属性增强**：随时间逐渐增强

### 3. 时间限制
- **基础时间**：10分钟（可配置）
- **倒计时显示**：在ActionBar显示剩余时间
- **警告时间**：最后60秒显示警告

### 4. 难度系统
- **随时间增强**：每分钟难度增加10%
- **最大倍率**：3倍最大难度
- **环境变化**：每5分钟触发环境变化（暴雨、夜晚等）

### 5. 奖励系统
- **击杀奖励**：基础50游戏币/只，随时间增加
- **里程碑奖励**：每5分钟额外100游戏币
- **排行榜**：记录玩家最佳成绩

## 🎮 测试步骤

### 步骤1：环境准备
1. 确保UpMobs插件已安装并启用
2. 确保挑战塔系统已配置并可用
3. 准备测试玩家账号

### 步骤2：解锁通天层
1. 使用测试账号通关挑战塔所有18层
2. 验证玩家进度是否达到19层（表示已通关）
3. 检查玩家是否满足解锁要求

### 步骤3：开始通天层挑战
1. 使用命令：`/upmobs tower sky start <挑战塔ID>`
2. 验证是否显示开始消息
3. 检查玩家是否被传送到挑战区域
4. 验证初始怪物是否生成（10只第四阶段升格怪物）

### 步骤4：基础功能测试
1. **怪物刷新测试**：
   - 击杀一只怪物
   - 等待5秒验证是否刷新
   - 检查怪物数量是否保持10只

2. **难度增强测试**：
   - 等待1分钟
   - 检查怪物属性是否增强10%
   - 验证难度倍率是否正确

3. **时间限制测试**：
   - 检查ActionBar是否显示倒计时
   - 等待到最后60秒验证警告显示
   - 验证10分钟时间到是否自动结束

### 步骤5：环境变化测试
1. **第一级变化（5分钟）**：
   - 等待5分钟
   - 验证是否触发暴雨天气
   - 检查是否显示环境变化消息

2. **第二级变化（10分钟）**：
   - 等待10分钟（需要延长测试时间）
   - 验证是否变为夜晚
   - 检查是否显示环境变化消息

### 步骤6：奖励系统测试
1. **击杀奖励测试**：
   - 击杀几只怪物
   - 验证是否获得游戏币奖励
   - 检查奖励金额是否正确（随时间增加）

2. **里程碑奖励测试**：
   - 存活5分钟
   - 验证是否获得里程碑奖励
   - 检查奖励消息是否正确显示

3. **排行榜测试**：
   - 完成挑战后
   - 验证统计数据是否保存
   - 检查控制台日志是否记录

### 步骤7：结束条件测试
1. **玩家死亡测试**：
   - 让玩家死亡
   - 验证挑战是否立即结束
   - 检查是否显示失败消息
   - 验证是否传送回入口位置

2. **时间结束测试**：
   - 存活10分钟
   - 验证挑战是否自动结束
   - 检查是否显示完成消息
   - 验证是否传送回入口位置

3. **手动结束测试**：
   - 使用命令：`/upmobs tower sky end`
   - 验证挑战是否立即结束
   - 检查是否显示结束消息

### 步骤8：状态显示测试
1. **状态信息测试**：
   - 使用命令：`/upmobs tower sky status`
   - 验证是否显示当前挑战状态
   - 检查信息是否包含：玩家、时间、击杀、奖励

2. **玩家状态显示测试**：
   - 在挑战中
   - 验证副标题是否显示状态信息
   - 检查信息是否包含：怪物数量、击杀数、奖励

## 🔧 配置调整

### 1. 调整怪物配置
```yaml
sky_tower:
  monster:
    base_count: 15                   # 增加怪物数量
    refresh_delay_seconds: 3         # 减少刷新延迟
    types:                           # 添加更多怪物类型
      - STAGE4_ZOMBIE
      - STAGE4_SKELETON
      - STAGE4_CREEPER
      - STAGE4_SPIDER
      - STAGE4_ENDERMAN
      - STAGE4_WITCH
      - STAGE4_BLAZE
```

### 2. 调整时间限制
```yaml
sky_tower:
  time_limit:
    minutes: 15                      # 增加时间限制
    warning_seconds: 120             # 增加警告时间
```

### 3. 调整难度系统
```yaml
sky_tower:
  difficulty:
    increase_per_minute: 0.15        # 增加难度增强速率
    max_multiplier: 4.0              # 增加最大难度倍率
    environmental_changes: false     # 禁用环境变化
```

### 4. 调整奖励系统
```yaml
sky_tower:
  rewards:
    base_per_kill: 100.0             # 增加基础奖励
    multiplier_per_minute: 1.5       # 增加奖励倍率
    milestone_interval_minutes: 3    # 减少里程碑间隔
    milestone_bonus: 200.0           # 增加里程碑奖励
```

## 🐛 故障排除

### 问题1：无法开始通天层挑战
1. 检查是否通关挑战塔所有18层
2. 检查通天层是否启用（配置中的enabled）
3. 检查玩家是否已经在通天层中
4. 验证挑战塔ID是否正确

### 问题2：怪物没有刷新
1. 检查刷新机制是否启用（refresh_on_death）
2. 检查刷新延迟配置是否正确
3. 验证怪物死亡事件是否正确处理
4. 检查怪物生成位置是否有效

### 问题3：难度没有增强
1. 检查难度增强配置是否正确
2. 验证时间计算是否正确
3. 检查怪物属性增强方法是否被调用
4. 验证难度倍率计算是否正确

### 问题4：奖励没有发放
1. 检查经济系统是否启用
2. 验证奖励计算是否正确
3. 检查击杀统计是否正确
4. 验证里程碑触发条件是否正确

### 问题5：环境变化没有触发
1. 检查环境变化是否启用
2. 验证时间计算是否正确
3. 检查环境变化方法是否被调用
4. 验证天气和时间设置是否正确

## 📈 性能考虑

### 1. 怪物管理
- 固定10只怪物，性能影响小
- 死亡后延迟刷新，避免瞬时压力
- 使用标签系统高效识别怪物

### 2. 任务调度
- 主任务每秒执行一次（轻量级）
- 怪物检查每2秒执行一次
- 状态更新可配置间隔（默认10秒）

### 3. 内存使用
- 每个玩家一个实例对象
- 怪物使用UUID索引，高效查找
- 统计数据按需保存

### 4. 优化建议
- 批量处理怪物刷新（如果数量很多）
- 缓存难度计算（如果时间固定）
- 异步处理奖励发放

## 🎯 设计优势

### 1. 可玩性
- 无尽挑战模式，重复可玩性高
- 随时间增强的难度曲线
- 丰富的奖励系统

### 2. 平衡性
- 固定怪物数量，避免数量爆炸
- 时间限制防止无限刷奖励
- 难度增强避免过于简单或困难

### 3. 扩展性
- 易于添加新的怪物类型
- 支持自定义环境变化
- 可集成其他插件系统

### 4. 用户体验
- 清晰的开始和结束流程
- 实时状态显示
- 丰富的视觉和音效反馈

## 🚀 快速开始命令

### 基础命令
```bash
# 查看通天层帮助
/upmobs tower sky

# 开始通天层挑战
/upmobs tower sky start <挑战塔ID>

# 查看通天层状态
/upmobs tower sky status

# 结束当前通天层挑战
/upmobs tower sky end
```

### 配置命令
```bash
# 启用/禁用通天层
/upmobs set sky_tower.enabled true

# 调整怪物数量
/upmobs set sky_tower.monster.base_count 12

# 调整时间限制
/upmobs set sky_tower.time_limit.minutes 15

# 调整基础奖励
/upmobs set sky_tower.rewards.base_per_kill 75.0
```
